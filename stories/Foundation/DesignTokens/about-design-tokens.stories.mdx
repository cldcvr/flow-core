# Design Tokens

<br />
<br />

## Overview

<f-divider></f-divider>

<p>
  Design tokens are the immutable values needed to construct and maintain a
  design system represented as data. They are not only a way to store
  information, but also a way to convey information.
</p>

## Color

<f-divider></f-divider>

In Flow, color system is broken into two parts

<ul>
  <li>
    <p>
      <a href="/story/foundation-design-tokens-base-color--page">Base colors</a>
    </p>
  </li>
  <li>
    <p>
      <a href="/story/foundation-design-tokens-system-color--page">
        System colors
      </a>
    </p>
  </li>
</ul>

## themes

<f-divider></f-divider>

<p>
  Themes are usually a collection of base colors, with each theme based on a
  specific primary background color (default surface)
  <br />
  Flow ships with two themes
</p>
<br />
<ul>
  <li>
    <p>
      Flow dark - The dark theme is automatically applied based on the user's
      preference of dark or light mode on their operating system.
    </p>
  </li>
  <br />
  <li>
    <p>
      Flow Light- The light theme is automatically applied based on the user's
      preference of dark or light mode on their operating system.
    </p>
  </li>
  <br />
</ul>
<p>Content and snippet for theme switching</p>
